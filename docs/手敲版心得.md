# AI 独立软件开发工作流

## 1. 基础方法论
- 自我约束
- 结构化构建

## 2. 计划与进度
- 维护 `PLAN.md`
- 维护 `PROGRESS.md`
- 每阶段必须有验收标准

## 3. 开发闭环
- 实现
- 测试
- review
- 更新进度
- commit

## 4. STOP HOOK
每次停止前检查：
- diff 是否符合目标
- 测试/lint 是否通过
- 进度是否更新
- 是否达到可提交状态

## 5. Git 检查点
- 阶段完成后 commit
- commit message 记录修改内容和验证方式
- 不自动执行 `git reset --hard`
- 不自动执行 `git push`

## 6. 测试驱动
- 功能完成必须有测试或验收步骤
- 测试通过后才能提交

## 7. 定期结构审查
- 文件是否过大
- 职责是否混乱
- 是否重复
- 是否需要重构

## 8. 日志系统
- 项目初期规划日志规范
- 关键路径和错误路径必须有日志
- 禁止记录敏感信息
